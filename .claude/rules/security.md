# セキュリティポリシー

## 機密情報

- APIキーは `.env`（`JQUANTS_API_KEY`）のみ。コード・テスト・ログ・例外メッセージ・標準出力・コミットに含めない
- ログ出力前にキー・トークンが混入しないことを確認する（HTTPヘッダのダンプ禁止）
- `.env`・DBファイル・取得データは `.gitignore` 済み。コミット前に `git status` で混入がないか確認する

## セキュアコーディング

- SQLは必ずパラメータ化する。文字列連結・f-stringでのSQL組み立て禁止
- YAMLの読み込みは `yaml.safe_load` のみ
- TLS検証の無効化（`verify=False` 等）禁止
- ファイル出力パスはユーザー入力をそのまま使わず検証する（パストラバーサル対策）
- サブプロセス起動時は `shell=True` を使わない

## 依存管理

- 依存パッケージの追加・更新時は `pip-audit` を実行し、既知脆弱性がないことを確認してからPRに含める
- 依存は最小限に。1機能のために大きなライブラリを足さない

## データの取り扱い

- 本ツールは個人情報を扱わない。銘柄・株価・財務データ以外を保存しない
- J-Quantsから取得した生データは再配布しない（リポジトリ・Confluence添付に含めない。Confluenceに載せるのは集計結果のみ）
