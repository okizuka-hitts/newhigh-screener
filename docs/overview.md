# newhigh-screener 要件概要

## サービスの目的

国内株式のスクリーニングを行うCLIツール。新高値(52週高値更新)銘柄の検出のみに特化する。

## 要件

- 国内株式スクリーニング。新高値銘柄の検出のみを対象とする(他のスクリーニング手法はスコープ外)
- 過去3か月分をさかのぼり、TOPIX 33業種ごとに財務・出来高などのフィルタパラメータをバックテストで変動させる(定期的な見直しを前提とする)
- データソースはJ-Quants API ライトプランのみ。APIキーは `.env` で指定
- 公開はソースコードのみ(GitHub)。取得データ・APIキーは含めない
- PythonのCLIツールとして実装する

## 制約

- J-Quants APIへのアクセス頻度はライトプランのアクセス上限の50%以内
- 株式分割を検知した場合(AdjustmentFactor ≠ 1)、データを再取得し分割補正を行う
- 実行予算はClaude Max x5プランの範囲内(開発プロセス上の制約)

## 用語定義

- **新高値**: 当日高値が過去52週間の高値を更新すること(詳細定義はNS-2で確定しREADMEに明文化)
- **33業種**: TOPIX 33業種分類(J-Quants銘柄一覧のSector33)

## 開発プロセス

- 開発はJIRA(NSプロジェクト)経由のEPIC駆動ループ。開発データ・テスト結果はConfluenceに蓄積する
- 各EPICの受け入れ基準が唯一の完了定義(スナップショット: `verification/acceptance/`)
- 仕様変更は人間の承認を経て本フォルダを更新してからコミットする
