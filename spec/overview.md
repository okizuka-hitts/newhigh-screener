# newhigh-screener 要件仕様

本フォルダは製品仕様の正である。実装・検証（Verifier）双方はここを参照する。仕様の変更は人間の承認を経てからコミットする。

## 1. 目的とスコープ

国内株式を対象に、**新高値（52週高値更新）銘柄の検出のみ**を行うCLIツール。
他のスクリーニング手法、リアルタイムデータ、検出結果の外部配信、Webサービス化はスコープ外。

## 2. 機能要件

### FR-1 データ取得（fetch）

- J-Quants API（ライトプラン）から以下を取得しローカルDB（SQLite）に蓄積する
  - 上場銘柄一覧（銘柄コード・銘柄名・33業種の業種コード/名称）
  - 日足四本値・出来高（直近3か月の全営業日、および52週高値判定に必要な過去52週分）
  - 財務データ（四半期・通期）
- `--verify` でデータ完全性を検査し、完全なら exit 0 / 欠損があれば exit 1 と欠損レポートを出力する

> **データソースのAPI仕様について**: 本仕様は J-Quants API の特定バージョンを前提にしない。
> ベースURL・エンドポイントのパス・認証方式・レートリミット・レスポンス形式は、
> 実装時点の最新の公式仕様（<https://jpx-jquants.com/ja/spec>）に準拠する。
> API がバージョン移行等で変わった場合は、最新仕様に合わせて実装・定数を更新する。

### FR-2 新高値検出（detect）

- 指定営業日に52週高値を更新した銘柄を検出する
- 出力項目：銘柄コード・銘柄名・33業種・終値・出来高・高値更新幅
- 出力形式：テーブル（標準出力）／ `--csv` でCSV
- 休場日・データ未取得日の指定は明確なエラーと exit code 非0

### FR-3 業種別パラメータ調整（tune）

- 絞り込みフィルタ（出来高倍率・財務条件）をTOPIX 33業種ごとに管理する（業種別YAML）
- 過去3か月分のバックテストでパラメータを調整する（定期的な再実行を前提とする）
- 結果（採用パラメータ＋成績指標）をConfluenceに記録する
- 同一入力に対して同一出力（再現性）

## 3. 非機能要件

- **NFR-1 レート制限**: J-Quants APIへのアクセス頻度はライトプランのアクセス上限の50%以内
- **NFR-2 分割補正**: 株式分割・併合の検知時（AdjustmentFactor ≠ 1）、該当銘柄の過去データを再取得し補正する
- **NFR-3 機密・データ管理**: APIキーは `.env` のみ。取得した生データ・DBはリポジトリに含めない（再配布しない）
- **NFR-4 検証可能性**: 取得パイプラインはテスト用データ注入フックを備える

## 4. 用語定義

- **新高値**: 当日高値が過去52週間の高値を更新すること（詳細はNS-2実装時に本仕様へ確定・追記）
- **33業種**: TOPIX 33業種分類（J-Quants上場銘柄一覧が提供する業種区分。具体の項目名は最新のAPI仕様に従う）
- **営業日**: 東京証券取引所の営業日
