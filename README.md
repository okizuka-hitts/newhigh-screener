# newhigh-screener

国内株式の新高値(52週高値更新)銘柄をスクリーニングするPython CLIです。
データソースは J-Quants API(ライトプラン)のみを使用します。

> **注意**: 本ツールは投資助言を行うものではありません。出力は情報提供のみを目的とし、投資判断は自己責任で行ってください。J-Quants APIから取得した生データはリポジトリに含まれません(利用規約に基づき再配布しません)。

## セットアップ

```bash
pip install -e ".[dev]"
cp .env.example .env   # JQUANTS_API_KEY を設定
```

## 使い方

```bash
screener fetch [--verify]          # データ取得(日次差分) / 完全性検査
screener detect --date 2026-07-10 [--csv out.csv]
screener tune --sector 3250 | --all
```

### 日次運用モード(fetch)

`screener fetch` は**日次差分モード**で動作します。毎営業日に実行することを前提に、
未取得の営業日分のみを短時間で取り込みます。

1. **初回(フルフェッチ)**: DBが空の状態で実行すると、52週高値判定に必要な過去52週分の
   日足・財務・上場銘柄一覧をまとめて取得します(全営業日ぶん)。
2. **2回目以降(差分)**: DBの最終取得営業日の翌日から当日までのうち、**まだ取得していない
   営業日分だけ**を取得します。取得済み期間へ重複リクエストは発行しません。
3. **休場日 / 最新済み**: 取得すべき新しい営業日が無いとき——休場日(土日・祝日・年末年始等の
   東証非営業日)に実行した場合や、すでに最新の営業日まで取得済みの場合——は、**データ取得
   リクエストを発行せず**にスキップし、その旨を表示して **exit 0** で終了します。

```bash
# 毎営業日、実行するだけ(差分のみ取得)
screener fetch

# 休場日に実行した場合の例(データ取得せず終了)
$ screener fetch
本日(2026-07-11)は休場日です。データ取得をスキップしました。
$ echo $?
0
```

**株式分割・併合**: 差分の取得中に分割・併合(調整係数 ≠ 1)を検知した場合、該当銘柄の
過去データを補正済みの値へ自動で上書きします(過去系列の遡及補正)。

**進行状況の表示**: 取得中は進行状況を表示します。

- 端末(TTY)では、パーセント・プログレスバー・残り推定時間を1行で更新表示します。
- CIやRoutine等の**非TTY環境では行単位のログ出力にフォールバック**し、進行情報が失われません。

**完全性検査**: `screener fetch --verify` はデータの完全性を検査します。完全なら exit 0、
欠損があれば欠損レポートを表示して exit 1 で終了します。

その他のコマンド(`detect` / `tune`)の仕様・新高値の定義は、該当EPIC実装時に本READMEへ追記されます。
